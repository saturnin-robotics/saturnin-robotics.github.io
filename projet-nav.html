Autonomous Navigation & SLAM

2025 TurtleBot 2 ROS 2 Humble / C++

This project focuses on deploying a complete navigation stack on a TurtleBot2 mobile robot using ROS 2. The goal was to achieve autonomous navigation in a cluttered environment using LiDAR data and odometry.

ROS 2 Nav2 Slam_toolbox Gazebo Rviz2

Technical Challenges

  • Sensor Fusion: Integrating wheel odometry with IMU data to reduce drift using Extended Kalman Filter (`robot_localization`).
  • Mapping: Generating a static map of the lab using `slam_toolbox` in async mode.
  • Navigation Tuning: Tuning the costmap inflation radius and the DWB local planner parameters to allow the robot to pass through narrow doors without collision.

System Architecture

The system runs on a Raspberry Pi 4 connected to the Micro-controller (Kobuki base). The high-level planning is done via the Nav2 stack. 

# Example: Launching the navigation stack
ros2 launch my_bot_nav bringup_launch.py map:=/maps/lab_map.yaml

Results

The robot successfully navigates from point A to point B with a success rate of 95%, avoiding dynamic obstacles (people walking) in real-time.
